# ГЛАС МОГА – Платформа за младежка активност

## Overview
**"ГЛАС МОГА"** е иновативна дигитална платформа, създадена да даде глас на младите хора и да ги насърчи към активно участие в обществения живот. Тя е пространство за тези, които искат не само да разбират проблемите около себе си, но и да изразяват мнението си и да предприемат реални действия за промяна. Платформата служи като мост между гражданските идеи и реалните резултати.

**Вашата основна задача** е **пълното реализиране на предоставения дизайн от Figma**, като същевременно интегрирате Frontend частта с готовия API сървър. Това включва:
- Разработка на потребителския интерфейс според визуалните изисквания.
- Динамично зареждане на сигнали, активности и истории на успеха.
- Реализиране на взаимодействията: подаване на сигнал, записване за участие, даряване и коментиране.
- Визуализиране на статистически данни за влиянието на общността.

## Tech Stack
- **Frontend**: React (Vite)
- **Routing**: React Router
- **Styling**: CSS Modules
- **Communication**: Axios
- **Backend API**: NestJS (Node.js) – локален сървър с автоматизирана документация.

## Prerequisites
Преди да стартирате проекта, трябва да имате инсталирани:
- **Node.js** (минимум 20.19.0, препоръчително LTS версия)
- **npm** (идва в комплект с Node.js)

Проверка на версиите в терминала:
```bash
node -v
npm -v
```

## Installation
В основната директория на проекта (там, където се намира този файл) изпълнете:

```bash
npm install
```
Тази команда ще инсталира автоматично всички необходими зависимости за **целия проект (frontend + api)**.

## Running the Application
Целият проект може да бъде стартиран едновременно с една команда от основната директория:

```bash
npm run dev:all
```
*(Можете да използвате и краткия вариант: `npm start`)*

Тази команда стартира автоматично:
- **Frontend**: [http://localhost:5173/](http://localhost:5173/)
- **Backend API**: [http://localhost:3000/api](http://localhost:3000/api)

### 💡 API Документация (Swagger)
За да ви бъде по-лесно при интеграцията на данните, сървърът разполага с документация (Swagger). Можете да я намерите тук:
👉 **[http://localhost:3000/swagger](http://localhost:3000/swagger)**

Swagger е вашият най-добър помощник, защото там можете да видите:
- Всички налични пътища (Endpoints), които можете да използвате.
- Как точно изглеждат данните, които сървърът връща.
- Какви параметри се очакват при изпращане на информация (напр. коментар или дарение).
- Можете дори да тествате всяка заявка директно от браузъра, преди да я напишете в кода.

Препоръчваме ви да започнете от него, за да се запознаете с възможностите на вашия backend.

## Project Structure
Проектът е организиран като **npm Workspace**, разделен на две основни части:

### 1. Frontend (`frontend-project-2026/`)
Основната логика се намира в папка `src/`:
- `pages/` – Основни страници (routes).
- `components/` – Функционални и преизползваеми компоненти.
- `api/` – Слой за комуникация с backend сървъра чрез Axios.
- `styles/` – Глобални стилове и дизайн токени (`tokens.css`, `base.css`).

### 2. API Server (`frontend-project-2026-api/`)
Локален сървър, изграден с NestJS, който управлява данните и бизнес логиката.

## Architecture Overview
Проектът следва модерна компонентно-базирана архитектура:
- **Data-driven pages**: Страниците управляват състоянието и зареждането на данни.
- **Centralized API**: Всички заявки са изнесени в `src/api/` за по-добра поддръжка.
- **Reusable Components**: Малките UI елементи са отделени от бизнес логиката.

## Styling Approach
Проектът използва **CSS Modules** за стилизиране.
Всеки компонент има собствен `.module.css` файл, което осигурява:
- **Локалност на стиловете** – стиловете на единия компонент не влияят на другия.
- **Избягване на конфликти** – няма нужда да се притеснявате за еднакви имена на класове.
- **По-добра модулност** – стиловете са тясно свързани с компонента си.

Глобалните стилове са разделени на:
- `tokens.css` – CSS променливи за цветове, шрифтове и разстояния.
- `globals.css` – Глобални основни стилове.

## Development Guidelines
При добавяне на нова функционалност следвайте тези правила:
1. Нови страници → `pages/`
2. Функционални блокове → `components/`
3. API логика → `api/`
4. Дизайн настройки → `tokens.css`

Стремете се да запазите архитектурния модел и да не пишете стилове директно в JS файловете (Inline styles).

## Notes & Troubleshooting
- **API Документация**: Всички налични ендпоинти са описани в Swagger интерфейса на сървъра.
- **Проблеми при стартиране**: 
  - Уверете се, че сте изпълнили `npm install` в основната папка.
  - Проверете дали портовете **5173** и **3000** не се използват от други приложения.
  - Ако сървърът не връща данни, проверете конзолата за грешки (CORS или мрежови проблеми).
